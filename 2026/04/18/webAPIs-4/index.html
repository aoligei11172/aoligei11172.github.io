事件流

2026/4/18

事件流

事件流指事件完整执行过程中的流动路径，分为三个阶段：

  1. 捕获阶段：从window对象向下传递到目标元素父节点，默认不触发监听器
  2. 目标阶段：事件到目标元素
  3. 冒泡阶段：从目标元素向上回溯到window，大多数事件监听在此执行
事件捕获

从DOM根元素去执行事件，代码：

DOM.addEventListener(事件类型,事件处理函数,是否使用捕获机制)

第三个参数 true / false 是否在捕获·阶段触发监听器，通常可不写，默认false

事件冒泡

当一个元素的事件被触发时,同样的事件将会在该元素的所有祖先元素中依次被触发，即当一个元素触发事件后,会依次向上调用所有父级元素的同名事件

阻止冒泡

因为默认有冒泡存在，容易导致事件影响父级元素，为把事件限制在当前元素，需要阻止事件冒泡。

前提：拿到事件对象

语法：事件对象.stopPropagation()可以阻止冒泡和捕获

阻止默认行为

event.preventDefault()

事件解绑
  1. on事件，直接用null覆盖
DOM.onclick = function(){}
DOM.onclick = null
//null直接覆盖

  1. addEventListener，必须用：

    removEventListener(事件类型,事件处理函数,{获取捕获或冒泡阶段})

    第三个参数可省略，前两个必须一致

匿名函数无法被解绑

事件委托

利用事件流的特征解决一些开发需求，减少注册次数，可以提高性能

原理：利用冒泡，给父元素注册事件，触发子元素时会冒泡到父元素，触发父元素事件

实现：事件对象.target.tagName 可以获得真正触发事件的元素

页面加载事件

1.事件名**load**

等待外部资源(图片、外联css和js)加载完毕时触发

监听页面所有资源加载完毕：

window.addEventListener('load',func());

也可针对某个资源绑定load事件

2.事件名**DOMContentLoaded**

当初始的HTML文档被完全加载和解析完成之后被触发

监听页面DOM加载完毕：

document.addEventListener('DOMContentLoaded',func());

页面滚动事件

滚动条滚动时持续触发

事件名**scroll**

获取滚动距离：

window.addEventListener('scroll',func() {
	const n = document.documentElement.scrollTop;
})
