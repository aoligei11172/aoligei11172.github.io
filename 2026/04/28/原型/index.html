原型

构造函数通过原型分配的函数是所有对象所共享的。

JavaScript规定,每一个构造函数都有一个prototype属性,指向另一个对象，即原型对象。

可以将公共的方法定义在prototype对象上

例：

    function Person(name,age){
        this.name = name;
        this.age = age;
    } //定义构造函数
    Person.prototype.sayname = function(){
        console.log(`${this.name}`);
    } //在原型变量上构造函数
	const A = new Person('张三',18); //定义对象
	A.sayname();  //输出 张三

constructor属性

prototype中的constructor属性，指向构造函数

    function Person(name){
        this.name = name;
    }
	console.log(Person.prototype.constructor === Person); //返回 true

用全新字面量修改prototype会让constructor丢失，故应该手动添加constructor指向原来的构造函数

    Person.prototype = {
        say : function (){
            console.log('say')
        },
        constructor : Person
    }
    console.log(Person.prototype.constructor 	=== Person) //返回true

__proto__

对象中都有一个属性__proto__指向构造函数的prototype，可以让对象使用原型对象的属性和方法

实例.__proto__ === 构造函数.prototype

    function Person(){}
	console.log(Person.__proto__ === 			Function.prototype); //输出true

原型继承

通过原型链实现属性和方法的共享与查找。

const Person = {
	eyes: 2,
}
function Man() {}
Man.prototype = new Person;
Man.prototype.constructor = Man;

原型链

原型链是JavaScript实现对象继承的核心机制，通过对象的内部属性[[Prototype]]形成链式结构，使对象能够继承其他对象的属性和方法。

当访问一个对象的属性时，JavaScript引擎按以下流程查找：

  1. 先在对象自身查找 - 检查对象自身的”显示属性”
  2. 若未找到，则通过__proto__在构造函数原型上查找
  3. 若仍找不到，继续向上查找原型的原型
  4. 直到找到Object.prototype（其__proto__为null，链的终点）
